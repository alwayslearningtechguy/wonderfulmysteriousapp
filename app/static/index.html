Overview

All endpoints are served under the base path /api/ and return JSON with appropriate HTTP status codes. The API is deployed via Docker and served by Uvicorn. Rate limiting is enforced globally — up to 10 requests per window per client. Exceeding the limit returns 429 Too Many Requests. Favorites are stored in-memory and reset on container restart.

⚡
Rate Limiting: All endpoints share a global limit of 10 requests per window. The 11th request returns 429 Too Many Requests. The limit resets automatically after the window expires.
GET /api/weather Weather ▶

Returns mock weather data for a given city. Ten US cities have detailed hardcoded data (Seattle, Miami, New York, Chicago, Los Angeles, Denver, Austin, Boston, Phoenix, Portland). Any other city name returns a sensible generic response rather than an error. Use the optional city query parameter to specify the location.

Query Parameters

NameTypeRequiredDescription
city string optional City name (default: Seattle)

Response Fields

city region temp_c temp_f condition humidity_pct wind_mph visibility_miles uv_index

Example Response — 200 OK

// GET /api/weather?city=Chicago
{
  "city": "Chicago",
  "region": "Illinois, US",
  "temp_c": 10,
  "temp_f": 50,
  "condition": "Windy with Clouds",
  "humidity_pct": 68,
  "wind_mph": 22,
  "visibility_miles": 9,
  "uv_index": 3
}

GET /api/insight Insight ▶

Returns a randomly selected insight from a pool of curated messages. Five topic pools are available: general, technology, productivity, leadership, and creativity. Unknown topics fall back to general. Each response includes an available_topics field listing all valid options.

Query Parameters

NameTypeRequiredDescription
topic string optional general · technology · productivity · leadership · creativity (default: general)

Response Fields

id topic insight available_topics

Example Response — 200 OK

// GET /api/insight?topic=technology
{
  "id": 482,
  "topic": "technology",
  "insight": "A good API is one that makes the wrong thing hard to do.",
  "available_topics": [
    "general", "technology",
    "productivity", "leadership", "creativity"
  ]
}
↗ Open /api/insight
GET /api/fortune Fortune ▶

Returns a randomly selected fortune from a pool of 12 curated entries. Each fortune includes a lucky number, a lucky color, and a piece of advice. No parameters are required. Results vary between calls.

Parameters

NameTypeRequiredDescription
No parameters

Response Fields

id fortune lucky_number lucky_color advice

Example Response — 200 OK

// GET /api/fortune
{
  "id": 37,
  "fortune": "The door you keep passing is the one worth opening.",
  "lucky_number": 8,
  "lucky_color": "Teal",
  "advice": "Act on the thing you have been deferring."
}
↗ Open /api/fortune
POST /api/submit Submit ▶

Accepts a structured JSON payload with a user string and a data object. The API validates the structure, sanitizes input (including script-like content), and returns 201 Created on success. Malformed JSON or missing required fields returns 422 Unprocessable Entity.

Request Body

FieldTypeRequiredDescription
user string required Identifier for the submitting user
data object required Arbitrary key-value payload to submit

Example Request & Response

// POST /api/submit
// Content-Type: application/json
{
  "user": "alice",
  "data": { "score": 99 }
}

// → 201 Created
{
  "status": "Created",
  "received": { "user": "alice", "data": { "score": 99 } }
}
